- The old styles/mobile.css file may remain in the repository; index.html no longer loads it.

WHAT CHANGED

1. True mobile presentation layer
- Mobile now has its own application shell and styling instead of shrinking the desktop workspace.
- Phones remain in the mobile application identity in portrait and landscape.
- Desktop and mobile continue sharing the same database, permissions, calculations, CRUD logic and reports.
- New dedicated stylesheet: styles/mobile-app.css.

2. Mobile login rebuilt
- Fits the real mobile viewport using dynamic viewport height and safe-area spacing.
- Prevents horizontal overflow and input zoom.
- Handles the on-screen keyboard more safely.
- Uses the real 59.svg and RE.svg in one coordinated horizontal logo lockup.
- Keeps Night mode as the fixed login theme.

3. Mobile navigation and workflows
- Fixed mobile header, bottom app navigation, More sheet and floating action button.
- Touch-first full-screen modals and forms.
- Safe spacing above the bottom navigation.
- No desktop sidebar on phones.

4. True mobile records
- Units, leases, tenants, payments, maintenance jobs, action items, audit activity and unit history use mobile cards.
- Portrait mode no longer squeezes the desktop tables.
- Landscape may expose expanded tables inside the mobile shell when useful.

5. Mobile charts
- Monthly maintenance trend is rebuilt to fit the phone width.
- Lease expiry shows the mobile summary without extending beyond the viewport.
- Page-level horizontal scrolling is blocked.

6. Users & Access mobile refinement
- User cards, role cards, grouped permissions, sessions and activity use a mobile-first layout.
- Permission controls remain readable and touch friendly.

7. Mobile document behavior
- Mobile actions use Save / Share PDF language.
- Official printing remains a desktop workflow.

8. Chrome property-image repair
- Automatically migrates old property cover paths stored in Chrome IndexedDB.
- Adds legacy filename aliases so old records remain compatible.
- Adds image fallbacks so a bad stored path cannot leave a broken-image icon.
- No local data reset is required.
