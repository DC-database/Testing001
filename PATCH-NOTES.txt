59 REAL ESTATE — PATCH v0.8.7
Base version: v0.8.6
Patch type: Dashboard unit-status reconciliation

PURPOSE
The Portfolio Overview previously displayed Total Units, Occupied, Booked and Ready Units, but units under inspection, maintenance, renovation or unavailable status were not represented in the main KPI row. This made a correct total appear incomplete to an executive viewer.

CHANGES
1. Added a permanent “Unavailable / In Progress” KPI card.
   - The card always remains visible, including when its value is 0.
   - At zero it shows “No units currently blocked.”
   - At a non-zero value it explains the included sub-statuses, such as Under Maintenance, Inspection, Renovation or Unavailable.

2. Added a Unit Status Reconciliation strip directly below the KPI cards.
   Example:
   373 Occupied + 11 Booked + 10 Ready + 1 In Progress = 395 Total

3. Added secure calculation fields to the dashboard snapshot:
   - blocked
   - underMaintenance
   - inspection
   - renovation
   - unavailable
   - otherBlocked

4. Added dashboard drill-down.
   - Clicking the new KPI or reconciliation item opens Properties & Units filtered to all blocked/in-progress units.
   - The filter includes Inspection, Maintenance, Renovation and Unavailable statuses.

5. Added responsive desktop and mobile presentation.
   - Desktop displays six KPI cards and automatically changes to a three-column layout on narrower screens.
   - Mobile gives the blocked KPI and reconciliation section a dedicated full-width layout.

DATA SAFETY
- No IndexedDB records are modified.
- No property, unit, tenant, contract, payment or maintenance data is deleted.
- This patch changes only calculations, filtering and presentation.
