59 REAL ESTATE PORTFOLIO IMPORT

Use one row per unit.

Required columns:
property_code, property_name, unit_number, unit_status, monthly_rent

Allowed unit_status values:
occupied, booked, vacant, maintenance, unavailable

For occupied or booked units:
tenant_name, contract_start and contract_end are required.
Use YYYY-MM-DD for dates.

For vacant, maintenance or unavailable units:
leave tenant and contract fields blank.

Do not rename column headers.
Property code + unit number must be unique.
Enter amounts as numbers only, without QAR or commas.

The importer separates the flat row into Property, Unit, Tenant and Contract records.
In production, the same validated importer will write through the Firebase provider.
